# Plan: Database-Backed Validation Bypass Rules

## Context

When an organization namespace is configured (e.g., prefix "FOOBAR", range 1000), the backend generates ATT&CK IDs like "FOOBAR-T1000". The ADM Zod schemas strictly enforce the standard format (e.g., "T####") and cannot be modified. This causes validation failures:

```
external_references.0.external_id is The first external_reference must match the ATT&CK ID format T#### or T####.###
```

**Goal**: Create a system that (1) provides CRUD for stateful validation bypass rules, (2) auto-manages namespace-related bypass rules, and (3) exposes a function `isErrorBypassed(error, stixType) → boolean` for callers to use.

As part of this effort, we also remove the legacy `validate-service.js`, `validate-controller.js`, `validate-routes.js`, `validate-paths.yml`, and `ERROR_TRANSFORMATION_RULES` from `validation-schemas.js`.

## New Files to Create

### 1. Mongoose Model — `app/models/validation-bypass-rule-model.js`
Pattern: `app/models/reference-model.js` (standalone `mongoose.model()`)

Fields:
- `fieldPath`: `[String]`, required — e.g., `["external_references", "0", "external_id"]`
- `errorCode`: `String`, required — e.g., `"custom"`
- `stixType`: `String`, required — e.g., `"attack-pattern"` or `"all"`
- `suppressError`: `Boolean`, default `true`
- `autoCreated`: `Boolean`, default `false` — marks entries auto-generated by namespace setup

Compound unique index on `{ fieldPath, errorCode, stixType }`.

### 2. Repository — `app/repository/validation-bypasses-repository.js`

### 3. Service — `app/services/system/validation-bypasses-service.js`
Pattern: `app/services/system/references-service.js`

CRUD methods:
- `retrieveAll(options)` — paginate via `BaseService.paginate`
- `create(data)` — `repository.save(data)`
- `retrieveById(id)` — `repository.retrieveById(id)`
- `deleteById(id)` — `repository.deleteById(id)`

Bypass logic:
- `isErrorBypassed(error, stixType)` — loads all rules from DB, checks if the error matches any active rule by comparing `fieldPath` (with `.map(String)` coercion for numeric indices), `errorCode`, and `stixType` (supports `"all"` or specific type). Returns `true`/`false`.

Namespace auto-management:
- `createNamespaceRules(stixTypes)` — bulk-create entries with `autoCreated: true` for each ATT&CK-ID-bearing STIX type, targeting `fieldPath: ["external_references", "0", "external_id"]`, `errorCode: "custom"`
- `removeNamespaceRules()` — `repository.deleteAutoCreated()`

## Existing Files to Modify

### 6. `app/services/meta-classes/base.service.js`

**6a.** Remove import of `processValidationIssues` from `validate-service.js` (line 25). Keep `getSchema` import but change it to import from `../../lib/validation-schemas` instead (since `validate-service.js` is being deleted).

**6b.** Rewrite `validateComposedObject(data)` (line 409) to be `async`. Instead of calling `processValidationIssues`, directly process Zod issues into the error format, then filter using `isErrorBypassed`:

```javascript
async validateComposedObject(data) {
  const empty = { errors: [], warnings: [] };
  if (!config.validateRequests.withAttackDataModel) return empty;

  const stixType = data.stix?.type;
  const status = data.workspace?.workflow?.state || 'reviewed';

  const schema = getSchema(stixType, status);
  if (!schema) return empty;

  const result = schema.safeParse(data.stix);
  if (result.success) return empty;

  // Convert Zod issues to error objects
  const allErrors = result.error.issues.map(issue => ({
    message: `${issue.path.join('.')} is ${issue.message}`,
    path: issue.path,
    code: issue.code,
    input: issue.input,
  }));

  // Filter out bypassed errors
  const validationBypassesService = require('../system/validation-bypasses-service');
  const errors = [];
  for (const error of allErrors) {
    const bypassed = await validationBypassesService.isErrorBypassed(error, stixType);
    if (!bypassed) {
      errors.push(error);
    }
  }

  return { errors, warnings: [] };
}
```
